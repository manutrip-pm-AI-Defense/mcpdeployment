## Directory Purpose

### `/src/mcpserver/`
Core MCP server implementation. All tools, resources, and prompts are defined in `server.py`. 

**Entry points:**
- `__main__.py` - stdio transport for local IDE integration
- `__main_remote__.py` - HTTP transport for remote deployment

Both entry points use the same `mcp` instance from `server.py`.

### `/scripts/scanners/`
Security scanner implementations using different analyzers:
- **yara_scanner.py**: Fast pattern-based scanning (no API key needed)
- **llm_api_scanner.py**: AI-powered analysis using OpenAI
- **cisco_api_scanner.py**: Cloud-based analysis via Cisco AI Defense

Run from project root: `uv run python scripts/scanners/yara_scanner.py`

### `/tests/`
Test suites and verification scripts.
- **test_scanner.py**: Validates scanner configuration and environment setup

### `/reports/`
Output directory for scan reports. All `*.json` files are gitignored. Reports are generated by scanner scripts.

### `/data/`
Static data files exposed via MCP resources. `sample_data.json` is read by the `demo://file/data` resource.

### `/docs/`
Project documentation:
- **methodology/**: Detailed scanner methodology and examples
- Scanner setup guides and troubleshooting

## Key Files

- **pyproject.toml**: Defines package metadata, dependencies, and CLI entry points
- **CLAUDE.md**: Development instructions for Claude Code
- **README.md**: User-facing documentation
- **.env.example**: Template for environment variables (copy to `.env`)

## Adding New Components

### New Tool/Resource/Prompt
Add to `src/mcpserver/server.py` using FastMCP decorators (@mcp.tool(), @mcp.resource(), @mcp.prompt())

### New Scanner
Create in `scripts/scanners/` following the pattern of existing scanners

### New Test
Add to `tests/` directory

### New Documentation
Add to `docs/` (general) or `docs/methodology/` (scanner-specific)
